A Utility to create safe debuggable harness for T-SQL stored procedures

Turn SQL Server stored procedures into runnable debuggable scripts for pre-production SQL Server - DML replaced with SELECT previews, variables traced with PRINT, no writes to real tables.

Python 3.10+ CLI · analyze & generate MIT License
Install from PyPI View on GitHub

What it does ?

Not a live debugger — a static harness generator you review, then run on a dev database. Output is stamped DEBUG HARNESS — DO NOT RUN ON PRODUCTION.

analyze

Inventory DML, TRY/CATCH, loops, SET statements, and line-level detail before you generate a harness.

generate

Build a debug script: stub real-table DML, add traces, strip deploy preamble, inline procedure parameters as DECLARE.

Enterprise-ready I/O

Auto-detect UTF-16 (SSMS Unicode) and Windows-1252 encodings. Optional step-by-step --log audit files.

How it works ?

Line scanning supplements sqlglot where TRY/CATCH and complex T-SQL hide DML from the AST.

Your .sql file CREATE PROC, TRY/CATCH, loops, MERGE, SSMS exports
→
sql-sp-harness Analyze or generate · comments & preamble optional
→
Debug script SELECT previews · PRINT traces · no CREATE PROC
⚠️ Always review generated scripts before running on a shared server. Dynamic SQL, cursors, and encrypted procedures are not fully rewritten.

Commands Available

Command Description
analyze Structural report — DML counts, TRY/CATCH, IF/WHILE, identified statements
generate Produce *_debug.sql harness with previews and traces
version Print installed package version

Common options

Flag Applies to Purpose
--trace-style print|raiserror generate Variable trace output (default: print)
--keep-comments generate Keep original -- and /* */ comments
--block-markers generate Insert [DBG] Step N before IF/WHILE
--log / --log-file both Timestamped step log for transforms and analysis
--encoding both Force utf-8, utf-16-le, cp1252, etc.
--no-stub-dml generate Traces only; leave DML unchanged

Install & Quick Start

Requires Python 3.10+ or newer.

# Step 1: Install the package
pip install sql-sp-harness

# Step 2: Verify the installation
sql-sp-harness version

# Step 3: Analyze / generate the procedure
sql-sp-harness analyze -i MyProc.sql
sql-sp-harness generate -i MyProc.sql -o MyProc_debug.sql

# Optional: step log beside input
sql-sp-harness generate -i MyProc.sql -o MyProc_debug.sql --log

Module invocation: python -m sql_sp_harness analyze -i MyProc.sql

Limitations

Pattern Behavior
Dynamic SQL Not analyzed or rewritten
Encrypted procedures No source available
Cursors Not rewritten
DDL inside proc Not stubbed
